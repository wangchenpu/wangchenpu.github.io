Maven插件相关

1、spring-boot-maven-plugin插件报错：repackage failed: Unable to find main class

打包时找不到主程序类(main方法所在类)

分析：该插件，只需要定义到可运行(有main方法)模块的pom文件中；其他模块中不需要

  • 一般，项目总pom文件，只负责管理依赖，所以不需要此插件
  • 项目中其他作为工具或类库的模块中，也不需要
  • 项目中的各个微服务模块，都可运行，需要此插件

解决：

1、项目总POM文件、

  • 去除spring-boot-maven-plugin插件
  • 打包方式：<packaging>pom</packaging>

2、项目需要运行的模块

  • 使用spring-boot-maven-plugin插件
  • 打包方式：<packaging>jar</packaging>

3、工具或类库的模块

  • 去除spring-boot-maven-plugin插件
  • 打包方式：<packaging>jar</packaging>

JavaWeb相关

1、请求转发下，刷新页面，表单重复提交问题

问题描述：

以登陆为例，在浏览器页面填写账号、密码然后点击登陆，账号密码正确登陆成功。
服务器端通过请求转发的方式响应浏览器一个页面，当在这个页面上点击刷新的时候，就会被提示是否重复提交表单，这个情况其实不应该出现

原因：请求转发是不改变浏览器的URL路径的；URL不改变则浏览器中的表单数据也还在，刷新就是重新提交一次

解决：用重定向的响应方式后浏览器就不会再保存原来URL的表单了

JSP页面，EL表达式无法解析

可能的原因有四个

  1. Application server in question doesn’t support JSP 2.0. （应用服务器不支持JSP2.0)
  2. The web.xml is not declared as Servlet 2.4 or higher. (web.xml中servlet版本没有声明在2.4以上）
  3. The @page is configured with isELIgnored=true. （页面上配置了<%@ page isELIgnored=”true” %> )
  4. The web.xml is configured with true in . (web.xml中显式地配置了忽略EL表达式）

Classes from the default package must not be referenced from JSP file
来自默认包的类，不能从JSP文件中引用

解决：给类添加一层包路径

下载功能，程序代码没问题，浏览器访问500错误

web项目的编译输出部署，资源文件是否编译到部署目录中

Jsoup解析html格式时，class之间有空格

<div class="result c-container ">

写成两个select ：doc.select(“div .result”).select(“.c-container”)

Git相关

将本地git仓库代码提交到GitHub上时，出现failed to push some refs to 'git@github.com:cq1415583094/MyBatis.git'

导致的原因是GitHub上存在一个readme.md的文件，解决

  1. git pull将远程与本地合并：git pull --rebase mybatis master
  2. 重新git push：git push -u mybatis master
